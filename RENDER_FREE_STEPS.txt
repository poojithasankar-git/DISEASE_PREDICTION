STEP 4: ADD ENVIRONMENT VARIABLES (CRITICAL!)
═══════════════════════════════════════════════════════════════

⚠️  WITHOUT THESE, KAGGLE DOWNLOAD WILL FAIL!

1. Scroll down to "Environment Variables"

2. Click "Add Environment Variable" button

3. First variable - KAGGLE USERNAME:
   ├─ Key: KAGGLE_USERNAME
   ├─ Value: [your kaggle username]
   └─ Scope: Build only
   
   (Get from: https://www.kaggle.com/settings/account
    Look for "Username" field)

4. Click "Add Environment Variable" again

5. Second variable - KAGGLE API KEY:
   ├─ Key: KAGGLE_KEY
   ├─ Value: [your kaggle API key]
   └─ Scope: Build only
   
   (Get from: https://www.kaggle.com/settings/account
    Click "Create New Token"
    It downloads kaggle.json file
    Open file and copy the "key" value)

✓ Both variables must be "Build only" scope!
✓ They are NOT available at runtime (privacy/security)

⏱️ Time: 2 minutes

🐛 IF TRAINING FAILS
═══════════════════════════════════════════════════════════════

Issue: Training takes too long (>24 hours)
Solution: Reduce EPOCHS in train.py to 5
  └─ Change: EPOCHS = 5
  └─ Redeploy (click Manual Deploy)
  └─ Training now takes ~35 minutes

Issue: Kaggle download fails
Solution: Check credentials
  ├─ Verify KAGGLE_USERNAME is correct
  ├─ Verify KAGGLE_KEY is not expired
  ├─ Go to https://www.kaggle.com/settings/account
  ├─ Create new token
  ├─ Update environment variables in Render
  └─ Redeploy

Issue: Out of memory error
Solution: Reduce batch size
  ├─ Change in train.py: BATCH_SIZE = 8 (from 16)
  ├─ Redeploy
  └─ Should work with less memory

Issue: Service crashes after starting
Solution: Check logs
  ├─ Go to Logs tab
  ├─ Look for errors
  ├─ Common causes: Model not found, import errors
  └─ Check if train.py completed successfully

═══════════════════════════════════════════════════════════════
🔄 WHAT HAPPENS IF SERVER RESTARTS?
═══════════════════════════════════════════════════════════════

Scenario: Server restarts/crashes

What Render does:
  1. Stops the service
  2. Restarts Flask app
  3. Model loads into memory again (takes 2-3 seconds)
  4. App is back online

What users experience:
  ├─ First request: Slow (2-3 sec wait)
  ├─ Model loads into memory
  └─ Subsequent requests: Fast (100-200ms)

⚠️ Model NOT retrained (it's already saved)
✓ Just loads the saved model from disk
